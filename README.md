# school21

Добро пожаловать в мой репозиторий с проектами, выполненными во время обучения в **Школе 21**. Здесь собраны практические задачи, решенные в рамках методологии "peer-to-peer" (равный равному).

## Направления и Проекты

### C
Фундаментальное программирование, работа с памятью, алгоритмами и структурами данных.
* **[SimpleBashUtils](./C/C3_SimpleBashUtils-2)** — Разработка утилит Bash по работе с текстом: cat, grep.
* **[String+](./C/C2_s21_stringplus-2-develop)** — Реализация библиотеки string.h с дополнениями.
* **[Decimal](./C/C5_s21_decimal-3)** — Реализация собственной библиотеки s21_decimal.h.
* **[Matrix](./C/C6_s21_matrix-3)** — Реализация библиотеки matrix.h.

### BSA 
Проектирование систем, сбор требований, моделирование бизнес-процессов.
* **[Decomposition](./BSA/BSA00_Decomposition.ID_1361439-1)** — Типовой жизненный цикл разработки ПО, типовой ролевой состав команды и основные функции каждой роли на этапах ЖЦ. 
* **[Stakeholders](./BSA/BSA01_Stakeholders.ID_1361441-1)** — Стейкхолдеры, примеры выявления стейкхолдеров и работа с ними. 
* **[Requirements](./BSA/BSA02_Requirements.ID_1361442-1)** — Требования к ПО, их виды, уровни связи и зависимости. Модели as is и to be, контекстная диаграмма, роли стейкхолдеров, их проблемы и потребности, бизнес-требования продукта. 
* **[HowToRequirements](./BSA/BSA03)** — Выявление требований и бизнес-процессов, построение функционально-ролевой модели, документация требований. 
* **[Domains](./BSA/BSA04_Domains.ID_1361447-1)** — Предметная область, выделение сущностей предметной области, информационная модель системы, проверка модели по CRUD. 
* **[Diagrams](./BSA/BSA05_Diagrams.ID_1361446-1)** — Разработка ТЗ, создание диаграмм UML (Use Case, Sequence, Component) и схем процессов в BPMN.
* **[BPMN](./BSA/BSA06)** — Основные модели и представления, диаграммы BPMN, области их применения и построение.
* **[UseStory](./BSA/BSA07_UserStory.ID_1361451-2/src)** — Создание UseStory и сопутстсующих им критериев приемки и тестовых сценариев. 
* **[UseCase](./BSA/BSA08_UseCase.ID_1361452-1)** — Сценарии и варианты использования. 
* **[ObjectsAndRoles](./BSA/BSA09_Objects_and_Roles_ID_1361454-Team_TL_stotchbe_75eb3452_9d46_4a76-2)** — Построение логической модели базы данных и словаря данных: сущностей, их атрибутов и диаграммы классов. 
* **[FunctionalRequirements](./BSA/BSA10_FunctionalRequirements_ID_1361456-1)** — Проектирование функциональной системы.
